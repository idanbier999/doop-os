# Security Policy

## Supported Versions

| Version | Supported |
| ------- | --------- |
| 0.1.x   | Yes       |

## Reporting a Vulnerability

**Please do NOT report security vulnerabilities through public GitHub issues.**

If you discover a security vulnerability, please report it responsibly by emailing:

**security@doop.dev**

Please include:

- A description of the vulnerability
- Steps to reproduce the issue
- The potential impact
- Any suggested fixes (optional)

## Response Timeline

- **Acknowledgment:** Within 48 hours of receiving your report.
- **Initial Assessment:** Within 5 business days.
- **Fix & Disclosure:** We aim to resolve confirmed vulnerabilities within 90 days. We will coordinate disclosure timing with you.

## Scope

This security policy covers the Doop Dashboard application and its dependencies.

## Recognition

We appreciate the efforts of security researchers who help keep Doop and our users safe. With your permission, we will acknowledge your contribution in our release notes.
